오토핏 개인정보처리방침

시행일 및 최종 업데이트: 2026년 5월 30일

오토핏(영문 서비스명: AutoFit Routine, 패키지명: com.motionroutine.app, 이하 "앱")은 사용자의 운동 루틴, 운동 기록, 체중, 인바디 기록, 눈바디/운동 사진을 사용자가 직접 기록하고 관리할 수 있도록 제공되는 운동 기록 앱입니다.

앱은 로컬 우선(Local-first) 정책을 따릅니다. 모든 사용자 데이터는 먼저 기기 내부에 저장되며, 사용자가 선택적으로 Google 로그인을 수행한 경우에 한해 일부 데이터가 클라우드 백업 서버(Supabase)와 동기화됩니다. 로그인하지 않은 사용자(게스트)의 데이터는 외부로 전송되지 않으며 오직 기기 내부에만 보관됩니다.

앱은 광고 SDK, 제3자 분석 SDK, 위치 추적 기능을 사용하지 않습니다.

2. 앱 권한 사용

  • 카메라: 운동 인증샷, 눈바디 사진 촬영
  • 사진/미디어 접근: 사용자가 선택한 사진을 앱에 가져오거나 갤러리로 저장
  • 알림: 휴식 종료 알림 등 로컬 알림 표시
  • 진동 및 오디오 설정: 세트·휴식 안내, 음성 안내, 진동 피드백
  • 백그라운드/화면 꺼짐 상태 동작: 운동 타이머와 휴식 알림의 안정 동작
  • 인터넷: Google 로그인 및 클라우드 동기화 (로그인한 경우에만 활성)
  • Google Play 결제: 정기결제 또는 인앱결제 처리

3. 처리위탁 및 제3자 제공

3.1 데이터 처리위탁

수탁자역할처리 항목보관 위치
Supabase, Inc. (미국 법인, AWS 인프라 사용) 클라우드 백업 및 동기화 서버 운영 1.2 의 동기화 대상 항목 대한민국 (서울 리전, AWS ap-northeast-2)
Google LLC Google 로그인 (OAuth 2.0) 인증 처리 Google 계정 이메일, 사용자 ID Google 인프라
  • Supabase 는 PostgreSQL 기반 BaaS 로, 행 단위 보안(Row Level Security) 정책에 따라 본인만 접근 가능하도록 격리 저장합니다.
  • 모든 통신은 TLS 1.2 이상으로 암호화됩니다.
  • Supabase 의 데이터는 저장 시 자동 암호화(AES-256)되며, 자동 백업이 적용됩니다.
  • Supabase 개인정보처리방침: https://supabase.com/privacy
  • Google 로그인 정책: https://policies.google.com/privacy

3.2 제3자 제공

앱은 다음 경우를 제외하고 개인정보를 제3자에게 제공하지 않습니다:

  • Google Play 결제 / Apple App Store 결제: 결제는 각 스토어를 통해 이루어지며, 앱은 결제 카드 정보를 직접 수집·저장하지 않습니다.
  • 사용자 직접 공유: 사용자가 데이터 내보내기 또는 공유 기능으로 백업 파일을 외부 서비스로 전달하는 경우. 해당 서비스의 개인정보처리방침이 적용됩니다.
  • 법령에 따른 요청: 수사기관 등의 법령에 근거한 정당한 요청이 있는 경우.

3.3 광고/분석 제3자 제공 없음

앱은 광고 SDK, 분석 SDK, 위치 추적 SDK를 사용하지 않습니다. 어떠한 사용자 데이터도 광고·마케팅·분석 목적으로 제3자에게 제공되지 않습니다.

4. 보유 및 이용 기간

4.1 로컬 데이터

  • 사용자가 삭제하거나 앱을 삭제하기 전까지 기기에 보관됩니다.
  • 인바디 기록은 앱 성능 관리를 위해 최대 500개까지 보관됩니다.
  • 눈바디/운동 사진은 무료 플랜 30장, Pro 플랜 최대 3,000장 한도 내 관리됩니다.

4.2 클라우드 데이터 (로그인 사용자만)

  • 사용자가 본인 계정으로 로그인 가능한 한 지속 보관됩니다.
  • 사용자가 앱 내 "클라우드 데이터 삭제" 기능을 사용하면 즉시 삭제됩니다 (로컬 데이터는 별도 유지).
  • 사용자가 Google 계정 연동을 해제하거나 본인 계정 삭제를 요청하면 30일 내에 영구 삭제됩니다.
  • Google 계정으로 90일 이상 로그인하지 않은 경우, 안내 후 클라우드 데이터를 삭제할 수 있습니다.

5. 개인정보 파기 방법

5.1 로컬 데이터 파기

  • 앱 내에서 루틴, 운동 기록, 체중, 인바디, 사진을 개별 삭제 가능합니다.
  • 앱 전체 데이터 삭제: 기기의 앱 정보 → 저장공간/데이터 삭제 또는 앱 삭제.
  • 삭제된 로컬 데이터는 복구할 수 없습니다.

5.2 클라우드 데이터 파기

  • 앱 내 Settings → 계정 → "로그아웃" 으로 클라우드 동기화 중단 (단, 클라우드 데이터는 보관됨 — 다시 로그인 시 복원).
  • 앱 내 "내 클라우드 데이터 삭제" 기능으로 Supabase 에서 즉시 영구 삭제 (예정 기능).
  • 또는 아래 문의 이메일로 계정 삭제 요청 시 30일 내 처리.

6. 이용자의 권리 (열람·정정·삭제·이동)

사용자는 「개인정보 보호법」 및 GDPR 등 관련 법령에 따라 다음 권리를 행사할 수 있습니다:

  • 열람권: 앱 내 모든 데이터를 화면에서 직접 확인할 수 있습니다.
  • 정정권: 앱 내에서 모든 기록을 수정할 수 있습니다.
  • 삭제권: 개별 항목 또는 전체 데이터를 앱 내에서 삭제할 수 있습니다. 클라우드 데이터 삭제는 위 5.2 절차를 따릅니다.
  • 이동권 (데이터 이식성): 앱 내 "백업 내보내기" 기능으로 본인 데이터를 JSON/ZIP 형식으로 다운로드할 수 있습니다.
  • 처리 거부권: Google 로그인을 하지 않으면 클라우드 처리가 일체 발생하지 않습니다. 로그아웃 후 게스트 모드로 계속 사용 가능합니다.

권리 행사 요청은 아래 문의 이메일로 접수할 수 있으며, 본인 확인 후 30일 내 처리됩니다.

7. 안전성 확보 조치

7.1 로컬 데이터 보호

  • 모든 데이터는 운영체제 앱 샌드박스 영역(/data/data/com.motionroutine.app/)에 저장됩니다.
  • 기기 백업(adb backup)을 통한 외부 추출이 비활성화되어 있습니다 (android:allowBackup="false").
  • 광고 식별자(AD_ID) 권한이 명시적으로 제거되어 있습니다.

7.2 클라우드 데이터 보호

  • 모든 통신은 TLS 1.2 이상으로 암호화됩니다.
  • 저장 시 AES-256 으로 자동 암호화됩니다 (Supabase 인프라).
  • 행 단위 보안(Row Level Security) 정책으로 본인 데이터만 접근 가능합니다. 다른 사용자나 익명 접근은 데이터베이스 레벨에서 차단됩니다.
  • 인증된 사용자 역할(authenticated)만 데이터 접근 권한이 부여되며, 익명 역할(anon)은 데이터 접근이 차단됩니다.

7.3 인증 보안

  • Google OAuth 2.0 (ID 토큰 검증) 으로 인증되며, 비밀번호는 앱이 직접 처리하지 않습니다.
  • 인증 토큰(JWT)은 기기 내 보안 저장소에 보관되며 1시간마다 자동 갱신됩니다.

8. 아동의 개인정보

앱은 만 13세 미만 아동을 주 대상으로 제공되지 않습니다. 만 13세 미만 아동이 앱을 사용하는 경우 보호자의 지도와 동의가 필요합니다. 보호자 요청 시 즉시 데이터 삭제 처리합니다.

9. 국외 이전

9.1 Google 로그인 인증

  • 인증 처리는 Google LLC (미국 본사) 의 글로벌 인프라를 통해 이루어집니다.
  • 처리되는 항목: Google 계정 이메일, 사용자 ID
  • 이전 목적: 로그인 인증
  • Google 정책: https://policies.google.com/privacy

9.2 클라우드 동기화 (Supabase)

  • 데이터는 대한민국 리전(AWS Seoul, ap-northeast-2) 에 저장됩니다.
  • Supabase, Inc. 는 미국 법인이지만, 데이터 자체는 국내에 머무릅니다.
  • 운영·관리는 Supabase 의 글로벌 팀이 담당하며, 시스템 관리 목적으로 접근할 수 있습니다.
